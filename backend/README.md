项目架构
1. 当前项目第一阶段并不需要保存用户信息，所以不需要数据库
2. 当前项目只提供 几个API接口分别是：
    1. 搜索接口
    2. 查看章节列表
    3. 查看章节内容
3. 由于只是工具，所以设置一个 token ,当 API请求的时候需要带上这个TOKEN才能正常使用，TOKEN 从环境变量中读取

爬虫架构设计：

会爬取不同的小说网站，每个网站的可能有极大的不同。所以需要针对不同的网站设计不同的爬虫。但是这些爬虫的表现要一致，即：
1. 搜索接口返回的结果要一致
2. 查看章节列表接口返回的结果要一致
3. 查看章节内容接口返回的结果要一致

在接口使用的时候，可以无视底层的实现细节
